# RegionSizeR (отрицательное биномиальное распределение, отношение частот)
[English](README.md) | Русский
Инструмент симуляции для планирования регионального размера выборки в
многорегиональном клиническом исследовании (MRCT) и оценки вероятности
подтверждения консистентности эффекта терапии в интересующем регионе. Конечная
точка - число событий, моделируемое отрицательным биномиальным распределением;
целевая мера - отношение частот (терапия против контроля); гипотеза - неменьшая
эффективность (non-inferiority); дизайн - повторные измерения (одну группу
замеряют дважды, до и после воздействия).
## Какую задачу решает
Когда препарат разрабатывают в рамках одного глобального исследования,
регуляторы отдельных стран (прежде всего Японии) требуют доказать, что эффект,
наблюдаемый в их регионе, согласуется с общим эффектом, а не является следствием
случайности на малой локальной выборке. ICH E17 формулирует это как оценку
консистентности эффекта между регионами; руководство MHLW Японии переводит
требование в критерий сохранения эффекта: регион должен удерживать фиксированную
долю общего эффекта, в наиболее консервативном варианте - не менее 50%.
Отсюда практический вопрос планирования: сколько субъектов нужно региону, чтобы
при реально работающем препарате исследование с высокой вероятностью выполнило
критерий консистентности? Инструмент отвечает на это методом Монте-Карло. Он
генерирует исследования при заданном эффекте, подгоняет модель и подсчитывает,
как часто одновременно проходит общий тест неменьшей эффективности и регион
удерживает не меньше целевой доли эффекта. Эта частота и есть вероятность
консистентности - величина, по которой определяют размер региона.
## Метод
Дизайн следует подходу сохранения эффекта из ICH E17 и руководства MHLW, опираясь
на рамку вероятности консистентности из работы Quan et al. (2010). Это
расширение на отрицательное биномиальное распределение и повторные измерения в
духе приложения RegionSizeR (Sun et al., 2024).
Консистентность измеряется как доля общего эффекта, удержанная в регионе:
- log-шкала: `(logRR_cn - log(nim)) / (logRR_main - log(nim))`
- шкала отношения частот: `(1 - RR_cn/nim) / (1 - RR_main/nim)`
где `_cn` обозначает интересующий регион, `_main` - общую популяцию, `nim` -
границу неменьшей эффективности. Смоделированное исследование считается
консистентным, если эта доля превышает порог `thd` (0.5 для японского правила).
## Два режима анализа
- **`correlated`** выбирает модель. `FALSE` рассматривает два измерения как
  независимые группы и подгоняет `glm.nb` с профильным доверительным интервалом.
  `TRUE` рассматривает их как внутрисубъектную пару и подгоняет отрицательную
  биномиальную GEE-модель (`geeM`) с неструктурированной рабочей корреляцией и
  робастными (сэндвич) стандартными ошибками на Wald-интервале.
- **`conservative`** выбирает знаменатель. `FALSE` делит на число сошедшихся
  моделей. `TRUE` считает каждую несошедшуюся модель провалом исследования, делит
  на полное число симуляций и выводит число провалов.
## Требования
R (>= 4.1) и следующие пакеты:
```r
install.packages(c("MASS", "simstudy", "geeM", "tidyverse", "flextable"))
```
`MASS` и базовый `stats` покрывают независимый режим. `simstudy` и `geeM` нужны
только для парного режима (`correlated = TRUE`). `tidyverse` и `flextable`
используются для таблицы результатов.
## Запуск
Откройте `RegionSizeR_negbin_RR.Rmd` и задайте два переключателя в блоке
`options`:
```r
correlated   <- FALSE   # FALSE = независимые группы, TRUE = парная GEE
conservative <- FALSE   # FALSE = только сошедшиеся, TRUE = провалы учитываются
```
Затем соберите отчёт (knit) в HTML. Один запуск даёт одну таблицу результатов.
Чтобы запустить без правки файла, соберите отчёт с параметрами:
```r
rmarkdown::render(
  "RegionSizeR_negbin_RR.Rmd",
  params = list(correlated = TRUE, conservative = TRUE)
)
```
Чтобы прогнать все четыре комбинации:
```r
grid <- expand.grid(correlated = c(FALSE, TRUE), conservative = c(FALSE, TRUE))
for (i in seq_len(nrow(grid))) {
  rmarkdown::render(
    "RegionSizeR_negbin_RR.Rmd",
    params = list(correlated = grid$correlated[i],
                  conservative = grid$conservative[i]),
    output_file = sprintf("result_%d.html", i)
  )
}
```
## Параметры
Задаются в блоке `params`.
| Параметр    | Значение                                                          |
|-------------|-------------------------------------------------------------------|
| `n`         | Число субъектов. 58 для независимого режима, 29 для парного (в обоих случаях 58 наблюдений). Устанавливается автоматически по `correlated`. |
| `ratio`     | Соотношение распределения между группами (только независимый режим).|
| `ratio_cn`  | Соотношение распределения внутри интересующего региона.           |
| `pct`       | Доля интересующего региона в общей выборке.                       |
| `pct_in`    | Доля субъектов региона, вошедших в глобальное исследование.       |
| `mean_trt`  | Средняя частота событий в группе терапии.                        |
| `mean_ctrl` | Средняя частота событий в группе контроля.                       |
| `nsim`      | Число симуляций.                                                 |
| `SigL`      | Уровень значимости.                                              |
| `thd`       | Порог консистентности (доля удержанного эффекта; 0.5 для Японии). |
| `od_ctrl`   | Сверхдисперсия Var(X)/mu, контроль. Должна быть больше 1.         |
| `od_trt`    | Сверхдисперсия Var(X)/mu, терапия. Должна быть больше 1.          |
| `nim`       | Граница неменьшей эффективности на шкале отношения частот.        |
| `hma`       | Направление гипотезы: 1 = большее значение хуже, 0 = лучше.       |
| `rho`       | Внутрисубъектная корреляция (только парный режим).                |
| `seed`      | Сид генератора случайных чисел.                                  |
## Вывод
| Поле              | Значение                                                       |
|-------------------|----------------------------------------------------------------|
| `n_global`        | Глобальный размер выборки.                                     |
| `n_chinese`       | Размер выборки интересующего региона.                          |
| `pct`, `pct_in`   | Доли региона в процентах.                                      |
| `conditional_p`   | Вероятность консистентности (log-шкала), условная на значимый глобальный тест. |
| `unconditional_p` | Вероятность консистентности (log-шкала), безусловная.          |
| `cond_p_red`      | То же, что `conditional_p`, на шкале отношения частот.         |
| `uncond_p_red`    | То же, что `unconditional_p`, на шкале отношения частот.       |
| `power_global`    | Вероятность значимости глобального теста неменьшей эффективности. |
| `power_cn`        | То же, ограниченное регионом.                                  |
| `threshold`       | Использованный порог консистентности.                          |
| `n_failed`        | Число несошедшихся симуляций (только консервативный режим).     |
## Воспроизводимость
Сид зафиксирован внутри драйвера симуляции, поэтому результаты воспроизводимы
между запусками. Переключатель `conservative` меняет
только пост-обработку (знаменатель), поэтому не затрагивает сами данные симуляции,
и консервативный и обычный режимы используют идентичные выборки.
